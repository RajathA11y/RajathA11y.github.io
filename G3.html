Guideline 1.3 Adaptable: Create content that can be presented in different ways (for example simpler layout) without losing information or structure

Purpose

To ensure that all information is available in a form that can be perceived by all users, for example, spoken aloud, or presented in a simpler visual layout. 

1.3.1 Info and Relationships I.E., Semantic markup is used to designate headings, lists (up, ol, and dl), emphasized or special text (strong, code, abbr, blockquote

Test Procedure for Headings

Screen Reader Test Procedure Expected result:
JAWS Use the keystroke “Insert + F6” which displays all the headings on the page in a list; ensure headings displayed in list view are matching the visual heading sequence; Verify that the headings are meaningful; JAWS should be able to detect headings 
NVDA Use keystroke Ins + f7 , use shift tab and select Headings from the category ensure headings displayed in list view are matching the visual heading sequence; Verify that the headings are meaningful; NVDA should be able to detect headings 
VoiceOver Press ctrl + options + u to open the rotor,, press the left or right arrow keys to cycle through the list of options until you find headings list ensure headings displayed in list view are matching the visual heading sequence; Verify that the headings are meaningful; VoiceOver should be able to detect headings 
Talkback On the webpage swipe down with three fingers until it announces headings list, Now swipe three fingers left to right for next heading and right to left for previous heading  Verify that the headings are meaningful; Talkback should be able to detect headings 

Test Procedure for Tables

Screen Reader Test Procedure Expected result:
JAWS USE the JAWS shortcut “T” to navigate to a table, Use the keystroke “Ctrl + Alt + arrow keys to move between the cells; Alternatively you can use insert + spacebar followed by t and use arrow keys Verify that jaws announces the column and row headings where ever applicable 
NVDA USE the NVDA shortcut “T” to navigate to a table, Use the keystroke “Ctrl + Alt + arrow keys Verify that NVDA announces the column and row headings where ever applicable 
VoiceOver Press ctrl + options + command + t for next table, Press ctrl + options + command + y for next column, Press ctrl + options + command + shift + y for previous column, Press ctrl + options + c to read the column heading, Press ctrl + arrow keys to navigate the rows and columns  Verify that VoiceOver announces the column and row headings where ever applicable 

1.3.2 Meaningful Sequence 

Screen Reader Test Procedure Expected result:
JAWS Use “Insert + down arrow” to make JAWS read the entire page Verify that the visual order and screen reader reading order are same 
NVDA Use “Insert + down arrow” to make NVDA read the entire page Verify that the visual order and screen reader reading order are same 
VoiceOver Use Control + Option + A to make VoiceOver read the entire page Verify that the visual order and screen reader reading order are same 
Talkback Draw the letter ‘l’ on the screen, in the screen select the option “read from next item”  Verify that the visual order and screen reader reading order are same 

1.3.3 Sensory Characteristics 

Instructions do not rely upon shape, size, or visual location For example:

  • Click the square icon to continue
  • Instructions are in the right-hand column
  • A beeping sound indicates you may continue
 

Test Procedure

No specific screen reader commands available, perform a Manual check to ensure the instructions are not relying upon shape, size, visual location, or sound
